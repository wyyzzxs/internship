# 答辩稿

> **目的**:8-10 分钟答辩,按角色分配讲稿,提前演练。

## 答辩流程

| 时间 | 内容 | 主讲 |
|---|---|---|
| 0-1.5min | 产品介绍 | 成员 F |
| 1.5-4.5min | 现场演示(3 个用例) | 成员 E |
| 4.5-6min | 技术架构 | 成员 A |
| 6-7min | 数据与知识库 | 成员 C |
| 7-8min | 难点与解决 | 成员 B |
| 8-10min | 提问环节 | 全员 |

## 成员 F(产品)讲稿(1.5min)

> 我们做的是「AI 智能旅游规划师」。目标用户是想自由行但懒得做攻略的人,典型场景是「武汉 3 天 1500 元文化游」。和普通攻略网站不同,我们会查实时天气、算预算、自动排时间轴。最酷的是支持对话修改——你说"把博物馆换成江滩",它就改了。这是基于课程第 7 天的 Agent 技术,真实可用,不是 PPT。

## 成员 E(演示)讲稿(3min)

**演示 3 个用例**:
- 用例 1:武汉 3 天 1500(快速展示,~30s)
- 用例 2:西安 3 天亲子(~30s)
- 用例 3:对话修改(重点,~60s)
- 展示地图 + 饼图(~60s)

## 成员 A(架构)讲稿(1.5min)

- 三层架构:Streamlit / FastAPI / Agent+工具
- 单 Agent + 7 个 FC 工具(景点检索 / 天气 / 预算 / 路线 / POI / Checklist / 日记)+ 反思
- RAG 用 ChromaDB + DashScope Embedding
- 多轮对话用 SQLite 存会话历史

## 成员 C(数据)讲稿(1min)

- 6 城 × 60-80 景点
- ChromaDB + text-embedding-v3
- 余弦相似度检索,top-5

## 成员 B(难点)讲稿(1min)

- 工具设计:LangChain @tool 自动生成 schema
- 失败兜底:Agent 死循环强制终止 / 工具调用失败自动重试
- 反思机制:自查行程,自动重新调度
