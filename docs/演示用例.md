# 演示用例

> 答辩演示使用的 3 个典型场景数据，覆盖基础功能、亲子场景、对话修改核心亮点。

---

## 用例 1: 武汉 3 天 2 晚 文化美食游（基础档展示）

**输入参数**:

| 字段 | 值 |
|---|---|
| 目的地 | 武汉 |
| 天数 | 3 天 2 晚 |
| 出发日期 | 2025-07-08 |
| 预算 | 1500 元/人 |
| 偏好 | 历史文化、美食探店 |
| 人群 | 朋友结伴 |
| 出发地 | 长沙 |
| 特殊要求 | 想看夜景，不吃辣 |

**期望输出亮点**:
- 时间轴包含: 黄鹤楼、湖北省博物馆、东湖绿道、户部巷、武汉长江大桥、江汉路步行街、古琴台
- 预算分配: 交通 300 + 住宿 500 + 餐饮 400 + 门票 200 + 其他 100
- 天气联动: 7/10 有阵雨，自动建议室内景点（博物馆）
- 地图展示: 7 个景点 marker + 路线连线

**演示话术**:
> "我们先看最基础的场景——三个大学生从长沙来武汉玩，预算 1500。系统会自动查天气、排时间轴、算预算。注意 7 月 10 号有雨，它自动把户外景点调到前两天，第三天安排博物馆。"

## 用例 2: 西安 4 天 3 晚 亲子历史游（进阶档展示）

**输入参数**:

| 字段 | 值 |
|---|---|
| 目的地 | 西安 |
| 天数 | 4 天 3 晚 |
| 出发日期 | 2025-07-12 |
| 预算 | 不限 |
| 偏好 | 历史文化 |
| 人群 | 亲子家庭 |
| 出发地 | 北京 |
| 特殊要求 | 孩子 8 岁，节奏不要太赶 |

**期望输出亮点**:
- 时间轴包含: 兵马俑、陕西历史博物馆、大雁塔、古城墙、大唐芙蓉园、回民街、华清宫
- 每天 3-4 个活动，节奏舒缓
- 亲子友好提示: 博物馆建议租讲解器、兵马俑建议请导游
- 住宿推荐: 钟楼附近亲子酒店
- 美食推荐: 肉夹馍、羊肉泡馍（可要求不辣版）

**演示话术**:
> "第二个场景是亲子游——带 8 岁孩子去西安。系统会考虑孩子的体力，每天只排 3-4 个活动，而且优先推荐有教育意义的景点，比如兵马俑和陕历博。"

## 用例 3: 对话式修改行程（核心亮点展示）

**前置**: 用例 1 已生成武汉行程

**对话修改记录**:

| 轮次 | 用户输入 | 系统响应 |
|---|---|---|
| 1 | "第三天加一个博物馆，预算加 200" | 自动加入湖北省博物馆，预算调整为 1700，时间轴重新排列 |
| 2 | "把户部巷换成昙华林，我想拍照" | 替换景点，更新地图路线，预算不变 |
| 3 | "预算降到 1200，怎么省？" | 建议: 住宿降档、减少一顿饭、去掉收费景点门票 |
| 4 | "帮我导出 PDF" | 生成完整攻略 PDF，含时间轴、地图、预算饼图 |

**演示话术**:
> "最厉害的是对话修改——你说换景点就换，说加预算就加，系统会实时重新计算时间和预算。最后还能一键导出 PDF 攻略，直接发到朋友圈。"
